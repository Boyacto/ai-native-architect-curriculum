# 진단 배터리 v1 — 초기 실력·격차 측정

커리큘럼 시작 전 학습자의 실제 수준을 측정하는 5세션 진단. 하루 1시간 × 5세션(1~2주).
결과는 역량 노드별 L레벨(미도달/L0/L1/L2+)로 기록하고, 이 결과가 Pass 2(첫 12개월 계획)의 입력이 된다.

## 운영 규칙

1. **진행자·채점자는 LLM(Claude)이 맡는다.** 세션 시작 시 "진단 S{N} 시작"이라고 요청하면 즉석에서 문항을 출제한다.
2. **실제 문항은 이 문서에 없다.** 사전 노출은 측정을 오염시킨다. 이 문서는 형식·규칙·루브릭만 정의한다.
3. **AI 금지 구간은 자기 신고제다.** 금지 구간에서는 자동완성·AI 어시스턴트를 끄고, 검색도 하지 않는다. 측정의 목적은 평가가 아니라 커리큘럼 보정이므로, 속이면 손해는 본인에게 돌아간다.
4. 각 세션은 **측정 → 즉석 채점·해설 → 기록** 순서로 60분 안에 끝낸다.
5. 결과는 `assessment/results/`(git 추적 제외)에 세션별로 기록한다.
6. 모르면 모른다고 답한다. "모른다"는 감점이 아니라 정확한 측정이다. 추측으로 맞힌 답이 커리큘럼을 잘못 보정한다.

## 세션 구성

### S1. 손코딩 리트머스 — AI 없이 코드가 나오는가

- 측정 도메인: A(CS·SW 기초) 중 구현 감각
- 형식: 소형 구현 문제 2개, 언어 자유(TypeScript/Dart 권장), **AI·검색·자동완성 금지**, 문제당 20분 제한
- 문제 수준: 자료구조 조작 + 상태 관리(예시 유형: 캐시, 파서, 큐 — 실제 문항은 즉석 출제)
- 남은 20분: 함께 리뷰 — 어디서 막혔는지(문법인가, 사고인가)를 구분해 기록
- 판정: 미도달(문법이 손에 없음) / L1(동작하지만 도움 필요) / L2(독립 완성) / L2+(엣지케이스까지 스스로 처리)

### S2. CS 기초 구두 진단 — 원리를 설명할 수 있는가

- 측정 도메인: A 중 DB 내부·네트워크·동시성·OS 기초
- 형식: 대화형 Q&A 40분. 정의 암송이 아니라 "왜 그렇게 동작하는가, 언제 문제가 되는가"를 묻는다. 답이 막히면 힌트를 주고 힌트 후 도달 여부를 구분 기록
- 영역: ① DB 인덱스·트랜잭션 ② HTTP/TCP와 요청의 생애 ③ 프로세스·스레드·async ④ 메모리·자료구조 비용
- 판정: 영역별 L0(용어만) / L1(힌트 받으면 설명) / L2(독립 설명 + 실무 연결)

### S3. 자기 코드 설명 — AI가 짠 코드를 책임질 수 있는가

- 측정 도메인: A·D + 실패 패턴 "AI 생성 코드를 이해하지 않고 승인" 직접 측정
- 형식: 본인 프로젝트(앵커 제품 권장)에서 진행자가 파일 2~3개를 무작위 지정. AI 없이 구두/텍스트로 설명한다:
  ① 이 코드는 왜 이 구조인가 ② 어떤 입력에서 깨지는가 ③ 다르게 설계한다면 어떤 대안과 트레이드오프가 있는가
- 판정: L0(무엇을 하는지만 앎) / L1(구조 설명 가능) / L2(취약점·대안까지 스스로 제시)

### S4. 설계 판단 — 대안을 만들고 비교할 수 있는가

- 측정 도메인: B(요구공학)·D(아키텍처)
- 형식: 본인 제품 맥락의 소형 설계 시나리오 1개(예시 유형: 신규 AI 기능의 동기/비동기 처리, 실패·비용 설계). 45분간 요구사항·제약 정리 → 대안 2개 → 트레이드오프 비교 → 선택과 근거를 작성. **AI 금지, 작성은 자유 형식**
- 판정: L0(대안 1개만) / L1(대안 2개, 비교 피상적) / L2(비용·실패·변경 가능성 축으로 비교하고 선택 근거 명시)

### S5. AI 시스템·제품 판단 — Deep Core 두 축의 현재 위치

- 측정 도메인: H(AI 시스템)·C(제품·지표)
- 형식: 구두 시나리오 2개, 각 25분
  - H: "당신 제품의 AI 출력이 좋은지 나쁜지 어떻게 판정할 것인가" — 평가 데이터셋·실패 복구 개념에 스스로 도달하는지 관찰
  - C: "수확 제품의 다음 개선을 무슨 데이터로 결정할 것인가" — 지표 설계·가드레일 개념 관찰
- 판정: 영역별 L0(감으로 답함) / L1(개념 언급, 구체화 못함) / L2(측정 가능한 방법을 설계함)

## 결과 기록 양식 (`assessment/results/S{N}.md`, 로컬 전용)

```
세션: S{N} / 날짜:
영역별 판정: (영역 — 레벨 — 한 줄 근거)
막힌 지점의 성격: (지식 부재 / 인출 실패 / 연습 부족)
커리큘럼 보정 메모:
```

## 종료 후

5세션 완료 시 진행자는 ① 도메인별 실측 L레벨 표 ② Pass 1 진단과의 차이 ③ 갱신된 curriculum_state를 출력하고, 그 결과로 Pass 2를 보정한다.
