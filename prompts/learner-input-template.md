# 학습자 입력 템플릿

커리큘럼 프롬프트의 `## 2. 학습자 입력과 진단` 섹션에 대응하는 입력 양식.
빈 항목은 그대로 두면 프롬프트가 합리적 가정을 세워 잠정안을 작성한다.
프롬프트 본문 뒤에 이 내용을 채워서 함께 붙여넣는다.

```
- 현재 학력과 컴퓨터과학 기초 수준:
- 현재 개발 역량과 주요 기술 스택:
- 제품·UX·기획 경험:
- 운영·인프라·보안 경험:
- AI 시스템 및 AI 보조 개발 경험:
- 팀·조직·고객 대응 경험:
- 현재 수행 중인 실제 프로젝트:
- 목표 직무와 목표 시장:
- 관심 산업 또는 전문 도메인:
- 주당 학습·프로젝트 가능 시간:
- 1년, 5년, 10년 목표:
- 취업·창업·외주·컨설팅 중 우선순위:
- 영어 및 기술 문서 독해 수준:
- 현실적 제약(비용, 장비, 지역, 조직 접근성, 건강, 일정 등):
```

## 세션 이어가기

새 세션에서 이어갈 때는 프롬프트 본문 + 위 입력 대신, 마지막 답변의 `curriculum_state` YAML 블록을 붙여넣으면 재진단 없이 이어서 진행된다.

## 분기 갱신 보고 양식

8~12주 사이클마다 아래를 보고해 진단을 갱신한다.

```
- 완료한 산출물과 링크:
- 완료하지 못한 항목과 이유:
- 막힌 지점(기술·시간·동기):
- 목표나 제약의 변경:
- 다음 분기에 쓸 수 있는 주당 시간:
```
